Expense Management Web Application

Splitsmart is a web application that will allow users to track and manage expenses. It shines in the aspect of lending or borrowing money from friends/family. For example say that you are splitting dinner between a group of friends one night and it would make it easier on the waiter to charge the meal all one card instead of multiple cards. You can open an expense and invite multiple people to the expense to pay you back.

I chose to create Splitsmart because I really wanted to create an application with a dynamic frontend. There is a lot of information on one page and it is all easily accessible. For instance if you click on an expense, that action renders a div below the expense to show who is included on the expense, who has paid, and comments on the expense of when someone may pay. The only time it feels like the web application truly serves another static page is when the user wants to pay for a specific expense.

One of my main achievements for this web application was the experience I got from implementing Graph QL query structure for my API. I feel Graph QL has made Splitsmart more organized and with the Apollo library made refetching data after a post a breeze giving the application a sense of real-time data updates.

Splitsmart can: Easily invite friends to expenses.

Post comments to keep your friends updated of when they should be expecting a payment.

Make payments through paypal or venmo for expenses or record a cash payment.
