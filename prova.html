Track your runs with Prova

I use the Nike running application a few times a week so when my group decided that they wanted to create one I was thrilled. My main contribution to the application was figuring out how the user would interact with the data that the user would be inputting into it. I decided to use the Nivo library for our graphs mainly because of how interactive their documentation is. There was not a lot of reading necessary which was nice for a change. All I had to do is find a map that I wanted to use play around with the settings on their site and copy and paste the code for the component into our app.

There were a lot of challenges, which felt like miniature leetcode problems, of converting the time, dates and calories into something more user friendly and meaningful from javascripts internal date module. My favorite thing that I did with the application was creating a static map after you create a route and displaying it in the stats page. It looks similar to the Nike run application's static map and I feel like in general just looks really cool.

Prova uses the Mapbox GL to easily create routes and provide directions to ensure you won't ever get lost.

Graphical representation of your progress running over a period of time is provided by the Nivo library.

Feel like you had a good run on a specific day? Rate your run by choosing one of three emojis.
